OpenrowDB **0.1.6** is the **definitive fix** for the `Task.sleep` crash that v0.1.3 and v0.1.4 attempted to address. It also includes TLS security improvements and web landing site fixes from a full QC audit.

### Fixed

- **`Task.sleep` crash (definitive)** — replaced all `Task.sleep` debounce calls with `DispatchQueue.main.asyncAfter` in workspace tab persistence, search, and column filter. The generation-counter approach in v0.1.3/v0.1.4 was insufficient — `swift_task_dealloc` aborts on macOS 26 / Swift 6 even without explicit cancellation.
- **Force-try crash paths** — replaced 4× `try!` in app startup fallback paths with proper `do-catch` error handling.
- **Info.plist version** — `CFBundleShortVersionString` was stuck at `0.1.0` in source; now correctly shows `0.1.6`.

### Security

- **TLS certificate verification** — `SSL=require` now uses system trust roots for full certificate verification on both PostgreSQL and MySQL drivers. `SSL=prefer` keeps opportunistic TLS for local/dev databases.

### Web landing site

- Added `rel="noopener noreferrer"` to all external links (reverse tab-nabbing protection).
- Added `og:image`, `twitter:image`, and `canonical` meta tags.
- Removed unused `gsap` dependency.

### CI/CD

- New Rust CI workflow (`cargo fmt`, `cargo clippy`, `cargo test`).
- SSH key cleanup step in site deployment.

### Install

```bash
curl -fsSL https://openrowdb.ryanprayoga.dev/install.sh | bash
```

```bash
curl -fsSL https://openrowdb.ryanprayoga.dev/install.sh | OPENROWDB_VERSION=0.1.6 bash
```
