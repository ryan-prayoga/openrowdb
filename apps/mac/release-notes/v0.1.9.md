OpenrowDB **0.1.9** turns the sidebar into a full schema browser and gives the query editor partial-run plus a live status bar — alongside two Liquid Glass UI fixes.

### Added

- **Schema browser sidebar** — a connection now expands to its databases → schemas → tables → columns, with inline row counts (`~` prefix for estimates) and column types. Browse any database on the server without switching the connection.
- **Sidebar SQL/DDL actions** — right-click context menus for: New / Rename / Drop / Truncate / Edit Structure / Export-as-SQL on a table; New / Drop / Export-as-SQL on a database; Copy CREATE statement; Copy Name / Copy Qualified Name on tables and columns.
- **Partial run** — ⌘↩ runs the selection, or the statement under the caret if nothing is selected; ⇧⌘↩ runs all. The Run menu exposes Run Selection / Run Current Statement / Run All Statements.
- **Editor status bar** — live `Ln · Col`, selection length, and char/line counts, like a native code editor.
- **Getting-started hints** — a fresh query tab shows the key shortcuts (run, format, autocomplete, cancel) in the results pane instead of an empty void.
- **Jump to error** — a failed result links straight to the offending token, moving the caret there.
- **Line-number gutter** — a hairline divider separates the line numbers from the code.

### Removed

- **Row-limit toolbar control** — the auto-`LIMIT` picker is gone; queries always run exactly as written.

### Fixed

- **Gutter seam** — the line-number ruler no longer bleeds a full-height vertical line behind the toolbar and results pane. The hosted `NSScrollView`'s ruler composited outside the editor frame; `compositingGroup()` + `clipped()` contains it.
- **Row inspector rounded corner** — the table viewer's right inspector rendered as an inset rounded-corner glass card (native macOS `.inspector`), cutting a rounded notch into the content. Replaced with a flush, square trailing pane via `safeAreaInset`.
