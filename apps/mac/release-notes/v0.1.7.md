OpenrowDB **0.1.7** fixes a sidebar bug where two connections with the **same name** would cross-wire their expand/collapse trees.

### Fixed

- **Sidebar tree cross-wire** — when two connections share the same name (e.g. two "chinook" connections to different databases), expanding or collapsing one would toggle the other instead. The expand/collapse state was tracked as a per-view `@State Bool` that leaked between sibling `ForEach` elements during SwiftUI `List` + `.sidebar` style cell recycling. Fix: lifted expand state to a centralized `Set<UUID>` in the parent sidebar view, passed down as `@Binding<Bool>` — each connection is now tracked by its UUID, not its visual position.
